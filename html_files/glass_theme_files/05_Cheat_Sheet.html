Networking & Identity

  • Security Groups: Stateful, instance-level firewall. Allow rules only.
  • NACLs: Stateless, subnet-level firewall. Allow & Deny rules. Needs ephemeral ports.
  • VPN: Always encrypted tunnel over internet (IPsec).
  • Direct Connect / ExpressRoute: Dedicated private fiber. Fast, high cost.
  • CIDR (10.0.0.0/16): /16 = ~65k IPs (VPC). /24 = 256 IPs (Subnet).
  • DNS Records: A (IPv4), AAAA (IPv6), CNAME (Alias), MX (Mail).
  • Standard Ports: 22 (SSH), 80 (HTTP), 443 (HTTPS), 3389 (RDP), 53 (DNS).

Terraform (IaC)

  • tf init: Downloads providers, initializes backend (.tfstate).
  • tf plan: Dry-run. Compares state to config, shows execution plan.
  • tf apply: Executes the plan against target infrastructure.
  • tf destroy: Deletes managed infrastructure.
  • State: Critical JSON file mapped to real infrastructure. Keep in S3 (Remote State) + DynamoDB (State Locking).
  • Modules: Reusable configuration blocks. Keep code DRY.
  • Drift: When manual UI changes break TF state. Fix: tf plan, rewrite TF, or tf import.

Kubernetes & Containers

  • Pod: Smallest deployable unit. Holds 1+ containers.
  • Deployment: Manages Pod scaling, updates, rollbacks.
  • Service: Exposes Pods (ClusterIP=internal, NodePort=dev, LoadBalancer=external).
  • Namespace: Logical cluster isolation (dev vs prod).
  • kubectl commands:
    kubectl get pods -n dev
    kubectl describe pod api-123
    kubectl logs api-123 -f

Frameworks & Resiliency

  • NIST CSF (5 Functions): Identify, Protect, Detect, Respond, Recover.
  • CIA Triad: Confidentiality (Encryption), Integrity (Hashing), Availability (HA Clusters/DR).
  • RTO: Recovery Time Objective (How fast must we recover?).
  • RPO: Recovery Point Objective (How much data loss is acceptable?).
  • DR Strategies: Pilot Light (Fast/Mid-cost), Warm Standby, Multi-Site Active-Active (Instant/Max-cost).

The STAR Method

S - Situation: Provide context (e.g., "At TefoLogic, our prod pipeline...").
T - Task: The problem (e.g., "...was deploying manually and risky.").
A - Action: WHAT YOU DID (e.g., "I wrote GitHub Actions & Terraform...").
R - Result: Impact (e.g., "Deployed 50% faster with zero config drift.").
