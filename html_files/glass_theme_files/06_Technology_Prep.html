D. Managed Kubernetes (EKS / AKS)

TMX requires K8s knowledge. You have Docker. Bridge the gap.

The Bridge Phrase: "I'm very comfortable packaging apps locally using Docker and Docker-compose. Since K8s is essentially orchestrating those containers at scale, I've spent the last month diving into control-plane architecture."

K8s ConceptCloud Context
EKS (AWS) / AKS (Azure)AWS/Azure manages the Master Nodes (Control Plane); you just manage Worker nodes and deployments.
Pod vs ContainerPods wrap containers. K8s scales Pods, not containers directly.
Service (LoadBalancer)Creates an AWS ALB/NLB implicitly to route external internet traffic to Pods inside the cluster.
ReplicaSet / HPAHPA (Horizontal Pod Autoscaler) reads CPU/Metrics and changes the Replica count automatically to match load.

E. Cloud Cost Management (FinOps)

TMX looks for efficiency alongside engineering.

  • Right-sizing: Monitoring CloudWatch CPU/RAM metrics to ensure we aren't using a t3.xlarge when a t3.medium will do.
  • Storage Tiers: Moving older logs from S3 Standard to S3 Glacier via Lifecycle Policies.
  • Compute Savings: Utilizing Spot Instances for stateless batch-processing, and Compute Savings Plans/Reserved Instances for static, 24/7 web apps.
  • Tagging: Mandating strict Terraform tagging (e.g., CostCenter=TradingDB) so billing is fully allocated.

F. Disaster Recovery (DR) & High Availability

Crucial for a Stock Exchange. Ensure you speak to RTO/RPO.

StrategyDescription
Multi-AZ (HA)Spreading EC2/RDS across multiple Availability Zones in a single region. Protects against datacenter loss.
Pilot Light (DR)Keeping minimal core infra running in Region B, and scripting the rest by Terraform in an emergency. RTO: Minutes to Hours.
Multi-Region Active-ActiveTraffic routed globally via Route 53 to Region A and Region B simultaneously. Databases replicate continuously. RTO: Zero. Expensive but needed for stock exchanges.

G. ARM Templates vs Terraform

They use Azure and AWS. Know the difference.

  • ARM Templates: Native Microsoft JSON (or Bicep format). Excellent for deep Azure integrations but locked to the Azure ecosystem.
  • Terraform: Provider-agnostic (HCL syntax). Can deploy AWS, Azure, and even Kubernetes resources via the same state file workflow. "I prefer Terraform as it's the industry standard for multi-cloud, which aligns with TMX's hybrid structure."
