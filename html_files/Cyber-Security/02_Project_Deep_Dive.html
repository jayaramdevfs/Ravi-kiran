PROJECT 1: SOC Threat Detection & Log Monitoring Lab

30-Second Elevator Pitch

"I built a fully functional SOC Monitoring Lab using the ELK Stack (Elasticsearch, Logstash, Kibana). I configured Filebeat on Windows and Linux endpoints to ship logs to a centralized Logstash server, where I applied grok filters for parsing. I then created Kibana dashboards to visualize authentication patterns and created alerts for suspicious activities like SSH brute-force and account password changes."

Architecture Diagram

[Endpoints: Windows/Linux] ↓ (Filebeat: Log shipping) [Logstash: Centralized Parser] ↓ (Grok Filters: Log normalization) [Elasticsearch: Storage & Indexing] ↓ (Kibana: Visualization & Alerting) [SOC Analyst Dashboard]

Technical Highlights

  • Log Correlation: Linked successful logins from unknown IPs to previous failed attempts to detect 'Low and Slow' brute force.
  • Normalization: Used Logstash Ruby filters to handle complex nested log structures from Windows Event Viewer.
  • Alerting: Set up thresholds for authentication failures (>5 in 1 min) to trigger high-priority alerts.
Q: What was the biggest challenge in setting up the ELK stack?
"Logstash Grok filtering. Some Windows events are very noisy and have unpredictable structures. I spent a lot of time testing patterns in the Grok Debugger to ensure 100% parsing accuracy for Event ID 4624 (Successful Login)."
Q: How would you scale this for TMX's multi-cloud environment?
"I would use a distributed ELK deployment in AWS using Amazon OpenSearch, or ship logs directly from CloudWatch/S3 using Lambda-based log shippers to maintain high throughput."

PROJECT 2: AWS Cloud Security Monitoring

30-Second Elevator Pitch

"Focused on securing the infrastructure layer of my York University projects, I implemented AWS IAM least-privilege policies. I monitored CloudTrail for 'UnauthorizedOperation' errors to detect potential lateral movement and configured VPC Flow Log monitoring to identify unusual outbound traffic patterns that might indicate data exfiltration."

Architecture Diagram

[AWS Infrastructure: EC2, S3, IAM] ↓ (CloudTrail / VPC Flow Logs) [CloudWatch: Log Storage] ↓ (Metric Filters: Pattern matching) [SNS: Alerting System] ↓ [Admin: Email/SMS Alerts]

Technical Highlights

  • IAM Hardening: Audited existing roles and reduced 'Administrative' access by 70%, using Policy Simulator for validation.
  • VPC Flow Logs: Created CloudWatch metric filters for 'REJECT' actions on Port 22 and Port 3389 to monitor for scans.
