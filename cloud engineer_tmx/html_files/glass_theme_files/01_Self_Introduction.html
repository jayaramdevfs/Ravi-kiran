SELF-INTRODUCTION & KEY ANSWERS

The scripts that will set the tone for the entire interview.

1. THE 60-SECOND INTRO (Use if they say: "Give us a brief overview")

"Good morning/afternoon! I'm Ravi Kiran Kambhampati, a cloud and cybersecurity professional based in Toronto. I'm currently completing my Post-Graduate program in Cloud Operations at York University, which follows my earlier Post-Graduate in Cybersecurity Operations — also from York. This dual specialization gives me a unique perspective: I don't just build cloud infrastructure, I build it with security baked in from the start.

Professionally, I worked as a Cloud Engineer Intern at TefoLogic, where I deployed AWS infrastructure using Terraform, configured IAM policies following least privilege, set up CI/CD pipelines with GitHub Actions, and managed security groups and CloudTrail monitoring — all in a production environment serving real clients.

I've also built two hands-on projects: a SOC Threat Detection Log Analyzer that acts as a mini-SIEM, and 'SOAR-Lite', an end-to-end security automation platform connecting Python, SQLite, and automated firewall blocks.

What excites me about TMX is the opportunity to work on Canada's critical financial infrastructure. My combination of security knowledge and cloud engineering skills means I can help build infrastructure that's not just scalable and reliable, but fundamentally secure — which is exactly what a stock exchange demands."

2. THE 90-SECOND EXTENDED (Use if they say: "Walk us through your resume")

[Deliver the 60-second intro, then smoothly seamlessly add:]

"...At TefoLogic, I got very comfortable with Infrastructure as Code. I wasn't clicking through the AWS console; I was writing Terraform modules for EC2 and VPC deployments, and using GitHub Actions to automate our testing and deployments. That experience taught me how to operate in a real DevOps culture.

On the certification side, I'm currently finalizing my prep for the CompTIA Security+ exam, and my immediate next goal for this year is the AWS Solutions Architect Associate, followed by the Certified Kubernetes Administrator (CKA).

I noticed the Cloud Engineer role at TMX emphasizes high availability, robust environments, and modernization across AWS and Azure. Given my background bridging CloudOps and CyberOps, I believe I can be highly effective in maintaining uptime while ensuring compliance and security are never compromised."

3. WHY TMX GROUP?

  • Critical Financial Infrastructure: "TMX isn't just another tech company; it's the financial engine of Canada. Building cloud infrastructure for TSX/Montreal Exchange means dealing with strict compliance (OSFI), zero-downtime requirements, and high stakes. That level of rigor appeals to my security operations background."
  • Cloud Modernization Journey: "I know TMX is actively optimizing its cloud footprint across AWS and Azure. I want to be part of a team that is building out resilient, modern infrastructure."
  • Security Importance: "In a financial ecosystem, security and infrastructure are inseparable. TMX's focus on trust aligns perfectly with my dual education in CyberOps and CloudOps."
  • Career Growth: "Working alongside senior engineers at TMX is the ideal environment for me to grow into a Senior Cloud Architect over the next few years."

4. WHY SHOULD WE HIRE YOU?

  • The Security+Cloud Bridge: "Most junior or mid-level cloud engineers treat security as an afterthought. I have a dedicated post-grad in it. I look at every Terraform script and AWS architecture through the lens of NIST and least privilege."
  • Proven Production Experience: "At TefoLogic, I dealt with real clients. I know what it means to push code through CI/CD, manage state in Terraform, and actively monitor infrastructure via CloudTrail."
  • Builder Mindset: "I don't just study theory. Getting my hands dirty building SOAR-Lite—writing Python to ingest logs, detect brute-force, and automatically trigger network blocks—shows I can automate solutions and write clean scripts."

5. 5-YEAR PLAN

Goal: "In 5 years, I see myself as a Senior Cloud Security Architect or Lead Cloud Engineer."

Path: "In the first year, I plan to deeply master TMX's specific AWS/Azure environment and acquire my AWS Solutions Architect and Kubernetes (CKA) certifications. By year 3, I want to be leading cloud modernization projects, heavily automating our deployments, and perhaps mentoring incoming junior engineers to reinforce a security-first engineering culture."

6. BIGGEST WEAKNESS

The Weakness: "Over-optimizing before shipping."

The Script: "Because of my security background, my tendency is to try and perfect a script or architecture to handle every possible edge-case before deploying it. I've realized this can slow down agile delivery. To mitigate this, I've started heavily utilizing the 'MVP' (Minimum Viable Product) mindset. At TefoLogic, I learned to push functional, secure baselines quickly via CI/CD, and then iterate and optimize in subsequent sprints. I use Jira/Trello to backlog the 'nice-to-have' optimizations so I don't get stuck in analysis paralysis."

7. QUESTIONS TO ASK THE INTERVIEWER (Pick 2-3)

  • "Given the high-availability requirements of a stock exchange, how is the Cloud team currently balancing the rollout of new features with strict reliability and regulatory compliance?"
  • "You mentioned a hybrid AWS/Azure environment. Is the goal to treat them as active-active for disaster recovery, or are specific workloads isolated to specific clouds?"
  • "What does the on-call, incident response rotation look like for this specific Cloud Engineer team?"
  • "If I start next month, what is the most critical pain point or project you'd want me contributing to in my first 60 days?"
  • "How closely does the Cloud Engineering team work with the dedicated InfoSec team at TMX?"

8. KEY PHRASES & BODY LANGUAGE

Use These Phrases (Yes) Avoid These Phrases (No)
"In my production experience at TefoLogic..." "In school we learned..."
"I implemented least privilege IAM..." "I made sure things were secure..."
"I utilized Terraform for Infrastructure as Code..." "I clicked through the AWS console to make..."
"My approach to resolving that incident was..." "I guess I would try to fix it by..."

Body Language Tips:

  • Eye Contact: Look exactly at the camera lens when speaking, not at the screen.
  • Hands: Keep hands visible occasionally. Use open hand gestures; it builds trust.
  • Pacing: When asked a highly technical question, pause for exactly 2 seconds before speaking. It shows thoughtfulness.
  • Nodding: Active listening when they speak. Smile. Engineers want to hire people they enjoy working with.
